## 3. アクター定義

### 3.1 人物（Person）

| アクター | 説明 | システムとの関係 |
|----------|------|------------------|
| **AI Developer** | AIコーディングツールを使用してコードを生成する開発者 | MCPクライアント（AI Coding Tool）を通じて、ライブラリのコンテキストを取得 |
| **Library Author** | OSSライブラリ/フレームワークの開発者・メンテナー | CLIを通じて自分のライブラリをYATAにインデックス登録 |

### 3.2 外部システム（External System）

| システム | 説明 | プロトコル | データフロー |
|----------|------|-----------|--------------|
| **AI Coding Tool** | GitHub Copilot、Claude、Cursor、Cline等のAIコーディングアシスタント | MCP (stdio/SSE) | 双方向：クエリ送信 → コンテキスト受信 |
| **GitHub** | ソースコードリポジトリホスティング | HTTPS / Git | 一方向：リポジトリ取得 (clone/API) |
| **Local File System** | ローカルストレージ | File I/O | 双方向：ソースコード読み取り、グラフ保存 |
| **OSS Libraries** | 解析対象のオープンソースライブラリ | - | 一方向：ソースコード取得 |

## 4. システム境界

### 4.1 YATA システムの責務

| 責務 | 説明 | 関連要件 |
|------|------|----------|
| **ソースコード解析** | AST解析によるエンティティ・関係性抽出 | REQ-KGC-001, REQ-KGC-002 |
| **知識グラフ構築** | グラフ構造の構築・永続化 | REQ-KGC-003, REQ-KGC-005 |
| **MCPプロトコル提供** | Tools, Resources, Promptsの提供 | REQ-MCP-001〜008 |
| **CLI提供** | インデックス作成・サーバー起動等 | REQ-CLI-001〜006 |
| **多言語サポート** | Python, TypeScript, JavaScript, Rust, Go | REQ-LANG-001〜005 |

### 4.2 システム外の責務（Out of Scope）

| 項目 | 理由 |
|------|------|
| AIモデルの推論 | MCPクライアント（AI Coding Tool）の責務 |
| コード生成 | MCPクライアント（AI Coding Tool）の責務 |
| ユーザー認証 | ローカル実行のため不要（REQ-NFR-007） |
| クラウドストレージ | ローカルファイルシステムのみ使用 |

## 6. 通信プロトコル

### 6.1 MCP通信

| 項目 | 値 |
|------|-----|
| **プロトコル** | Model Context Protocol (MCP) 1.0 |
| **トランスポート** | stdio (ローカル), SSE (リモート) |
| **メッセージ形式** | JSON-RPC 2.0 |
| **認証** | なし（ローカル実行前提） |

### 6.2 GitHub通信

| 項目 | 値 |
|------|-----|
| **プロトコル** | HTTPS (REST API), Git over HTTPS |
| **認証** | Personal Access Token (PAT) - オプション |
| **レート制限** | 60 req/h (未認証), 5000 req/h (認証時) |
